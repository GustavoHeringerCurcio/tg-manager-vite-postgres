# Refactoring Frontend Plan — Telegram Botflix v2

## 1. Overview

### Current State

- Single `Dashboard.tsx` page doing everything in one two‑column grid.
- No routing library — all state toggled via `useState`.
- Zero UI library — everything is raw Tailwind classes inlined in JSX.
- No dark‑mode toggle (dark is hardcoded).
- No sidebar, no proper header, no layout component.
- Auth is a simple login form in `App.tsx`.

### Target State

- **Full visual rebrand** — zero visual resemblance to the old cyan-on-navy sci-fi look. Target aesthetic: professional dark admin dashboard (Vercel/Linear style).
- **shadcn/ui** as the sole component library for a consistent, professional design system.
- **react‑router‑dom** for client‑side routing (v7 already installed).
- **Persistent sidebar layout** across all authenticated pages.
- **Pages separated by domain** (dashboard, bots, bot detail, transactions, interactions, etc.).
- **Dark‑mode toggle** via `next-themes`.
- Toast notifications, skeleton loaders, proper empty states.

## 4. Route Design

```
/login                              → LoginPage           (no sidebar, standalone)
/manager                            → ManagerPage         (bot card grid)
/manager/new                        → CreateBotPage       (full create form)
/manager/:botId/dashboard           → BotDashboardPage    (stats overview)
/manager/:botId/messages            → BotMessagesPage     (message flow editor)
/manager/:botId/remarketing         → BotRemarketingPage  (remarketing editor)
/manager/:botId/transactions        → BotTransactionsPage (paginated transactions)
/manager/:botId/interactions        → BotInteractionsPage (paginated interactions)
*                                   → NotFoundPage        (404)
```

**Auth guard:** `<ProtectedRoute>` wraps all `/manager/*` routes. Redirects to `/login` if no token / 401.

**Layout wrapper:** All `/manager/*` routes render inside `<AppShell>` which provides the sidebar + header + `<Outlet />` for child content.

## 5. Component Architecture

### 5.1 `App.tsx` (Root)

```tsx
<BrowserRouter>
  <ThemeProvider defaultTheme="dark" attribute="class">
    <Routes>
      <Route path="/login" element={<LoginPage />} />
      <Route element={<ProtectedRoute />}>
        <Route element={<AppShell />}>
          <Route index element={<Navigate to="/manager" replace />} />
          <Route path="/manager" element={<ManagerPage />} />
          <Route path="/manager/new" element={<CreateBotPage />} />
          <Route path="/manager/:botId/dashboard" element={<BotDashboardPage />} />
          <Route path="/manager/:botId/messages" element={<BotMessagesPage />} />
          <Route path="/manager/:botId/remarketing" element={<BotRemarketingPage />} />
          <Route path="/manager/:botId/transactions" element={<BotTransactionsPage />} />
          <Route path="/manager/:botId/interactions" element={<BotInteractionsPage />} />
        </Route>
      </Route>
      <Route path="*" element={<NotFoundPage />} />
    </Routes>
    <Toaster />
  </ThemeProvider>
</BrowserRouter>
```

### 5.2 `ProtectedRoute.tsx`

```tsx
export default function ProtectedRoute() {
  const { authenticated, loading } = useAuth();
  if (loading) return <LoadingSkeleton />;
  if (!authenticated) return <Navigate to="/login" replace />;
  return <Outlet />;
}
```

## 6. Page Designs

### 6.1 `LoginPage.tsx` (`/login`)

- Extracted from current `App.tsx` login form.
- Uses shadcn `Card`, `Input`, `Button`, `Label`.
- Minimal design — centered card on dark gradient background.
- On success → `navigate("/manager")`.
- Already authenticated → `Navigate to="/manager"`.

### 6.2 `ManagerPage.tsx` (`/manager`)

- Fetches all bots via `useBots()` hook.
- Renders `BotCardGrid` — responsive CSS grid (1 col mobile, 2 col tablet, 3‑4 col desktop).
- Last card is a "Create New Bot" card (`+` icon, dashed border, different style).
- Each `BotCard` shows the bot summary and navigates on click.
- Loading state: `Skeleton` cards.
- Empty state: "No bots yet. Create your first bot!" message + CTA button.

### 6.2.1 `BotCard.tsx`


- Clicking the card body → `navigate("/manager/:botId/dashboard")`.

- Card uses `hover:border-primary/50 transition-all` for interactivity.

### 6.3 `BotDashboardPage.tsx` (`/manager/:botId/dashboard`)

- Fetches bot + stats via `useBotDetail(botId)`.
- Page title: bot name + `StatusBadge`.
- Stats cards in responsive grid (3‑5 columns):

| Card | Icon | Content |
|------|------|---------|
| Total Users | `Users` | `stats.totalUsers` |
| Total Interactions | `Activity` | `stats.totalInteractions` |
| Messages Sent | `Send` | `stats.messageCount` |
| Callbacks | `MousePointerClick` | `stats.callbackCount` |
| Checkout Clicks / Payments | `CreditCard` | `stats.checkoutClicks` |
| Daily Active Users | `UserCheck` | `stats.dailyActiveUsers` |

- Each stat card uses `<Card>` with icon in top‑left, value large, label muted below.
- Loading: `Skeleton` cards.
- Quick links row: buttons to navigate to Messages, Transactions, Interactions for this bot.

### 6.4 `BotMessagesPage.tsx` (`/manager/:botId/messages`)

- Refactored message flow editor.
- Fetches bot data via `useBotDetail(botId)`.
- **Bot settings section** (top): name, token, checkout amount — using shadcn `Input` + `Label`.
- **Message flow section**: list of collapsible `MessageStepCard` components.
- Each step is a `Card` with `Collapsible`:
  - **Header (always visible)**: step #, admin title, `Badge` for type (Text/Audio/Video), delay in seconds, buttons count.
  - **Expanded content**: type select, text/textarea, media URLs input, delay input, buttons editor.
  - **Actions**: move up/down, duplicate, remove.
- "Add Step" button at bottom.
- Save button → calls `api.updateBot` → toast success/error.
- Loading: `Skeleton`.

### 6.5 `BotRemarketingPage.tsx` (`/manager/:botId/remarketing`)

- Refactored remarketing editor.
- **Config section**: enable/disable `Switch`, interval (hours), initial delay (hours), max sends.
- **Message sequence**: same `MessageFlowEditor` component reused.
- **Bulk import**: `Textarea` + "Import" button for `URL | Caption` format.
- Save button → calls `api.updateBot`.
- Loading: `Skeleton`.

### 6.6 `BotTransactionsPage.tsx` (`/manager/:botId/transactions`)

- Paginated `Table` with shadcn `Table`, `TableHeader`, `TableBody`, `TableRow`, `TableCell`.
- Columns: Date, User, Amount, Status, Payment Method, PIX Code, Checkout URL.
- Status uses `Badge` variants.
- Checkout URL: clickable link (opens in new tab).
- PIX Code: truncated with `Tooltip` to reveal full code.
- Pagination: `<Button>` group (Previous / Page X of Y / Next) at bottom.
- Loading: `Skeleton` rows in table.
- Empty state: "No transactions found for this bot."
- Error state: alert with retry.

### 6.7 `BotInteractionsPage.tsx` (`/manager/:botId/interactions`)

- Paginated `Table`.
- Columns: Date, User, Type (`Badge`), Direction (IN/OUT), Content preview (truncated).
- Filters bar (top):
  - Type: `Select` dropdown (All, Message, Callback, etc.).
  - User ID: `Input` with debounce.
  - Date range: two `Input type="date"` (or use a `Popover` calendar in phase 2).
  - Apply filters button.
- Expandable row: click row → `Sheet` or inline expand showing raw JSON payload.
- Pagination same as transactions.
- Loading: `Skeleton` rows.
- Empty state: "No interactions found."

### 6.8 `CreateBotPage.tsx` (`/manager/new`)

- Full‑page form (not a modal) at `/manager/new`.
- Uses same `BotForm` component as `BotMessagesPage` — shared logic.
- Bot name, Telegram token (required for creation), checkout amount, message flow, remarketing.
- Save → calls `api.createBot` → on success, toast + navigate to `/manager/:newBotId/dashboard`.
- Cancel → navigate back to `/manager`.

### 6.9 `NotFoundPage.tsx`

- Centered 404 with `Card`, icon, message, "Go to Dashboard" button.

## 7. Hooks Design

### 7.1 `useAuth.ts`

```ts
export function useAuth() {
  // State: authenticated, loading, error
  // On mount: check token in localStorage, verify with api.bots()
  // login(password): setToken, verify, update state
  // logout(): clearToken, update state, navigate to /login
  // refresh(): re‑verify current token
  // Returns: { authenticated, loading, error, login, logout }
}
```

### 7.2 `useBots.ts`

```ts
export function useBots() {
  // State: bots (Bot[]), loading, error
  // fetchBots(): GET /api/bots → set bots
  // createBot(payload): POST /api/bots → refetch
  // updateBot(id, payload): PUT /api/bots/:id → refetch
  // setStatus(id, status): PATCH /api/bots/:id/status → refetch
  // deleteBot(id): DELETE /api/bots/:id → refetch
  // Returns: { bots, loading, error, createBot, updateBot, setStatus, deleteBot, refresh }
}
```

### 7.3 `useBotDetail.ts`

```ts
export function useBotDetail(botId: string | undefined) {

  // Returns: { bot, stats, loading, error, refresh }
}
```
